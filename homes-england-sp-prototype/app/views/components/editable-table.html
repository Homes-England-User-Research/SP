Editable table

A GOV.UK-styled table where specified cells contain inputs, allowing users to enter or update multiple related values simultaneously without navigating away from the page.

When to use this component

Use the editable table on data-entry screens where users need to update multiple related values in a single session — particularly where users are accustomed to an IMS-style (Investment Management System) workflow. It works best for structured datasets such as units by tenure type or quarterly expenditure by category.

When not to use this component

For single-value data entry, use a standard GOV.UK text input on its own page following the GDS one-thing-per-page pattern. The editable table is an intentional deviation from that pattern, justified only for SP professional data-entry users. Do not use it in public-facing flows.

Relationship to the GOV.UK Design System

The editable table extends the GOV.UK table component by placing govuk-input elements inside <td> cells. This pattern is not in the GDS component library. It is an intentional departure from GDS one-question-per-page guidance, justified by SP user research showing that data-entry users found single-column page-per-question patterns significantly slower for multi-row structured record entry.

Accessibility

  • WCAG 1.3.1 Info and Relationships: every input has an aria-label combining the row header and column header, e.g. "Shared Ownership - Units at Start on Site". Screen readers announce the full context of each cell.
  • Table structure: <th scope="col"> on all column headers; <th scope="row"> on tenure type row headers. This creates the correct programmatic relationship between headers and data cells.
  • WCAG 4.1.3 Status Messages: auto-calculated "Units Remaining" cells carry aria-live="polite" so screen readers announce recalculated values when they change without moving focus.
  • WCAG 1.4.1 Use of Colour: deficit values show a govuk-tag--red tag with the text "Deficit" — colour is not the sole indicator of the deficit state.
  • Focus styles: GOV.UK yellow focus outline is preserved on all inputs. Padding is reduced (not removed) to maintain compact rows.
  • Keyboard: Tab moves between inputs in DOM order (left to right, top to bottom), which matches the visual reading order.
  • Tested with: NVDA + Chrome (Windows). [Further testing to be added]

Research

SP users previously entered unit data into IMS, which presented all tenure types on a single dense screen. When a single-question-per-page prototype was tested, SP users found it significantly slower and expressed a strong preference for seeing all tenure types simultaneously. The editable table pattern was introduced to replicate the IMS workflow while maintaining GDS component standards and WCAG 2.1 AA compliance. [Quantitative research data to be added]
