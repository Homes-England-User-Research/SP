Stat card

Displays a single key metric with a label, value, optional trend indicator and optional supporting text. Used on overview and dashboard pages.

When to use this component

Use the stat card on overview and dashboard pages where users need to scan multiple key performance indicators quickly. It works best when you have between 2 and 6 metrics to display and each metric has a single headline value.

When not to use this component

Do not use the stat card where a single piece of information is better expressed inline in a paragraph or in a summary list. For general public-facing services, prefer the standard GDS summary list or inset text patterns.

Relationship to the GOV.UK Design System

The stat card is a new pattern with no direct equivalent in the GOV.UK Design System. It extends the visual language of govuk-panel to create a scannable KPI tile. The trend indicator uses GOV.UK colour tokens (green for positive, red for negative) but always pairs colour with a text label and directional symbol to satisfy WCAG 1.4.1 Use of Colour.

Accessibility

  • WCAG 1.4.1 Use of Colour: trend direction is conveyed by a directional arrow symbol (⇧ / ⇩) and a text percentage, not colour alone. The colour modifier (green/red) reinforces meaning but is not the sole indicator.
  • Screen reader label: the <article> element carries a combined aria-label of the form "[Label]: [Value], [Trend]" so screen readers announce the full stat in one statement.
  • Trend icons: arrow characters are wrapped in aria-hidden="true" spans. Screen reader text equivalents ("Up" / "Down") are provided via govuk-visually-hidden spans.
  • Keyboard: no interactive controls; the card is display-only. Focus passes through naturally.
  • Colour contrast: value text (#0b0c0c on #ffffff) exceeds WCAG AA 7:1. Trend text uses GDS green (#00703c) and red (#d4351c) which pass WCAG AA 4.5:1 against white.
  • Tested with: NVDA + Chrome (Windows). [Further testing to be added]

Research

SP programme managers reviewed multiple screens of tabular data in IMS to understand overall programme performance. Dashboard-style metric tiles were identified as a faster scanning pattern for headline KPIs. The stat card was built to reduce the time needed to orientate on an overview page.
