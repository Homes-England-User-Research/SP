Year tabs

A tab component for navigating between financial year panels, built to the WAI-ARIA Authoring Practices Guide tab pattern. All panels remain visible without JavaScript.

When to use this component

Use the year tabs component when the same data structure repeats across multiple financial years and users need to switch between them frequently — for example, comparing quarterly expenditure across programme years.

When not to use this component

Do not use tabs for general page navigation — use breadcrumbs or a contents list instead. Do not use tabs to hide information that users will frequently need to compare — consider a table or stacked sections instead.

Relationship to the GOV.UK Design System

The GOV.UK Design System does not currently include a tab component. This component follows the WAI-ARIA APG tab pattern using role="tablist", role="tab", role="tabpanel", aria-selected, aria-controls and aria-labelledby. It degrades gracefully — all panels are visible in the no-JavaScript state.

Accessibility

  • WAI-ARIA tab pattern: uses role="tablist", role="tab" and role="tabpanel" with correct aria-controls / aria-labelledby relationships.
  • Keyboard — tab selection: Left and Right arrow keys move focus between tabs and activate them. Home and End jump to first and last tabs. Enter and Space are handled natively by the <button>.
  • Roving tabindex: the selected tab has tabindex="0"; all others have tabindex="-1". Tab key moves into and out of the tablist as a single stop.
  • Graceful degradation: without JavaScript all panels are visible as stacked sections. No content is hidden from users without JS.
  • WCAG 1.3.1 Info and Relationships: the programmatic relationship between each tab and its panel is expressed via aria-controls and aria-labelledby, not just visual proximity.
  • Tested with: NVDA + Chrome (Windows). [Further testing to be added]

Research

SP users frequently need to compare data across multiple financial years without leaving the current page. Navigating to separate pages for each year was identified as a friction point during IMS usage. Year tabs provide fast in-page switching while keeping the current year prominent.
