Multi-column form

A form layout that places related inputs side by side in a two-column grid, reducing vertical scroll for data-dense entry screens used by SP professional users.

When to use this component

Use the multi-column form layout on screens where multiple closely related fields - such as start and end dates, agreed and forecast unit counts, or X and Y coordinates - are always entered together by professional data-entry users who understand the relationships between the fields.

When not to use this component

Do not use multi-column form layout for general public-facing services - use the standard single-column GDS form layout. The GDS single-column guidance exists because most users find single-column forms easier to complete, especially on mobile. This component is only justified for SP professional users on data-entry screens where research has confirmed a clear preference.

Relationship to the GOV.UK Design System

This is an intentional deviation from GDS question page guidance which recommends a single-column layout. The deviation is justified by SP user research showing that professional data-entry users found single-column layouts significantly slower for structured record entry. The layout satisfies WCAG 1.3.2 Meaningful Sequence because CSS Grid is used for visual layout only - DOM order is logical regardless of the visual arrangement.

How it looks in context

The multi-column form layout is used on programme profile and site detail entry pages where SP users enter multiple paired values in a single session.

Accessibility

  • WCAG 1.3.2 Meaningful Sequence: CSS Grid is used for visual layout only. DOM order is: label → input (left column), label → input (right column), repeating per row. Tab order matches visual left-to-right, top-to-bottom reading order.
  • WCAG 1.4.10 Reflow: the two-column layout collapses to a single column at the tablet breakpoint (768px) via a CSS Grid media query. No horizontal scrolling is required at 400% zoom.
  • Labels: every input has its own <label> element - labels are never shared, omitted or replaced by placeholder text.
  • WCAG 1.3.1 Info and Relationships: related fields are grouped inside a <fieldset> with a <legend> to provide group context.
  • Focus styles: GOV.UK yellow focus styles are fully preserved on all inputs.

Research

During usability testing with SP programme staff, participants entering paired data values (start/end dates, agreed/forecast units) reported that the single-column layout required excessive scrolling and made it harder to check that paired values were consistent. A two-column layout was tested and preferred in sessions with 4 out of 5 participants. [Full research note to be added]
