Date picker

The standard GOV.UK date input (day/month/year fields) with a supplementary calendar button that reveals a visual month grid. The calendar is a progressive enhancement - the base date input works fully without JavaScript.

When to use this component

Use the date picker on date fields where users may not know the exact date and benefit from a visual calendar to orientate themselves - for example, selecting a programme milestone date or a site visit date relative to a known week.

When not to use this component

For well-known dates (such as date of birth, or a date the user has in front of them), use the standard GDS date input without the calendar enhancement. The calendar adds cognitive overhead that is not needed when users already know the date they want to enter.

Relationship to the GOV.UK Design System

This component is a progressive enhancement of the GOV.UK date input component. The base component is the standard GDS date input and works fully without the calendar enhancement. The calendar is supplementary only. This follows the GDS progressive enhancement guidance.

Accessibility

  • Progressive enhancement: the standard GDS date input (day/month/year fields) works fully without JavaScript. Users who disable JS or whose JS fails see the standard input with no degradation.
  • Calendar toggle button: aria-label="Open date picker", aria-expanded reflects open/closed state, aria-controls references the calendar popup id.
  • Calendar grid: uses role="dialog" with aria-modal="true" and aria-label. Day buttons have descriptive aria-label values ("15 March 2025"). aria-selected="true" marks the currently selected date.
  • Keyboard - calendar navigation: Arrow keys move between dates. Enter or Space selects the focused date and populates the inputs. Escape closes the calendar and returns focus to the toggle button. Tab moves focus naturally out of the calendar, closing it.
  • WCAG 2.3.3 Animation: calendar open/close transition is suppressed when prefers-reduced-motion: reduce is active.
  • WCAG 1.4.3 Contrast: today's date and selected date indicators use GDS colour tokens that meet 4.5:1 contrast on white.

Research

SP users entering milestone dates for sites needed to know which dates fell within a reporting quarter. A plain date input required them to switch to a calendar application to find the relevant date. The visual calendar enhancement was prototyped to test whether it could eliminate this context switch. [Research to be added]
