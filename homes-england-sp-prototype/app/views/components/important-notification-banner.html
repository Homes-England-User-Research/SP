Important notification banner

An extension of the GOV.UK notification banner for time-sensitive messages that include an inline call-to-action link, used on dashboard pages where users have a pending action with a deadline.

When to use this component

Use the important notification banner on dashboard or overview pages when a user has a time-sensitive action to complete - for example, a quarterly return due in fewer than 5 days, or an approval request awaiting their response.

When not to use this component

For general information that is not time-sensitive, use the standard GOV.UK notification banner. Do not use the important notification banner for errors - use the error summary instead.

Relationship to the GOV.UK Design System

This component extends the GOV.UK notification banner. It adds an inline call-to-action link and an aria-live="polite" region so dynamically injected banners are announced to screen reader users. The "Important" title variant uses govuk-notification-banner--success classes as a base but with additional SP-specific modifier classes.

Example

{# Important notification banner. role="region" with aria-label identifies this as a landmark for screen readers. aria-live="polite" ensures dynamically inserted banners are announced. The CTA link text is descriptive ("Submit your Q2 return") not generic ("click here"). #}

Important

Your Q2 quarterly return is due in 3 days (31 October 2025). Submit your Q2 return

Accessibility

  • WCAG 1.3.1 Info and Relationships: role="region" with aria-label="Important notification" creates a named landmark so screen reader users can navigate directly to it.
  • WCAG 4.1.3 Status Messages: aria-live="polite" announces the banner content to screen readers when it is dynamically inserted into the page without moving focus.
  • WCAG 2.4.6 Headings and Labels: the banner heading ("Important") is a proper <h2> within the landmark region.
  • WCAG 2.4.4 Link Purpose: the CTA link text is fully descriptive ("Submit your Q2 return") and unambiguous out of context. Never use "click here" or "read more".
  • WCAG 1.4.1 Use of Colour: the banner uses a blue left border and "Important" heading - the urgency is conveyed by the heading text and deadline date, not by colour alone.
