Overview page

Build analysis

This page documents how the Overview page was built, how it aligns with the GOV.UK Design System and WCAG 2.1 AA, all custom departures with their justifications, and predicted accessibility tool outcomes.

{# ── Overview ───────────────────────────────────────────────────────── #}

Overview

The Overview page is a dashboard for housing programme managers at Homes England. It provides navigation sections, key metrics and data insight charts. It follows the IMS (Investment Management System) pattern of data-dense information display rather than the standard GDS one-question-per-page pattern, which is not appropriate for this use case.

The page has two data states — MVP (no data) and Mature (full data) — controlled by a prototype toggle in the footer. All GDS components used on the page are used without modification unless listed under Custom departures below.

{# ── GDS alignment ──────────────────────────────────────────────────── #}

GDS alignment

  • Tabs — govuk-tabs with data-module="govuk-tabs", unmodified. Keyboard navigation (arrow keys between tabs) provided by govuk-frontend JavaScript. Used to organise four data insight chart panels.
  • Button — govukButton(), unmodified. Build Analysis link (secondary variant) and prototype controls Apply button.
  • Details — govukDetails(), unmodified. Prototype controls toggle.
  • Select — govukSelect(), unmodified. Data state selector (prototype controls only).
  • Table — govukTable(), unmodified. Data tables inside insight blocks, always rendered first in DOM before the chart SVG.
  • Breadcrumbs — govukBreadcrumbs(), unmodified. Build Analysis page only.
  • Spacing — govuk-spacing() tokens used throughout. No custom px values.
  • Page width — govuk-width-container, 960px max-width, unmodified.
  • Grid — govuk-grid-row and govuk-grid-column-full. Standard GDS 12-column grid, unmodified.
{# ── Custom departures ──────────────────────────────────────────────── #}

Custom departures

{# ── Stat card ──────────────────────────────────────────────────────── #}

Stat card (metric card)

Departure: Bordered card displaying a single metric with optional trend indicator and coloured top border, used in the Key information section. Displayed in a full-width 2×3 CSS grid.

Justification: No GDS equivalent exists. The closest official pattern is govuk-summary-list, designed for form review screens, not dashboard metrics. Using it here would be a significant usability regression for users scanning programme health at a glance. The coloured top border convention is used across HMRC and ONS internal tools.

GDS principles: Aligns with Design Principle 4 (do the hard work to make it simple) and Principle 6 (this is for everyone).

Accessibility: Trend indicators use shape (▲/▼) + colour + aria-label — never colour alone (WCAG 1.4.1). Alert state (red border) reserved for actionable blockers only. GDS green (#00703c) on white achieves 7.1:1; GDS red (#d4351c) on white achieves 4.5:1 — both exceed WCAG 1.4.3 minimum. Windows High Contrast Mode handled via @media (forced-colors: active) to restore borders and badge visibility.

{# ── 2×3 metric card grid ──────────────────────────────────────────── #}

2×3 metric card grid

Departure: Six stat cards arranged in a 2-row × 3-column CSS grid using .stat-card-grid. No GDS grid component supports card-based layouts.

Justification: The GDS grid system is column-based (govuk-grid-column-*), not card-based. A CSS grid provides equal-width cards that reflow to single column on mobile (640px breakpoint). The 3-column layout matches the information density SP users expect from IMS.

Accessibility: Reflows to single column at 400% zoom (WCAG 1.4.10 Reflow). No horizontal scroll at any zoom level.

{# ── Two-column navigation grid ────────────────────────────────────── #}

Two-column navigation grid

Departure: Five navigation sections arranged in a 2-column CSS grid using .nav-section-grid. Halves the vertical space compared to a single-column stack.

Justification: Reduces page length significantly. Matches how the GDS Design System itself groups related content in its documentation. Reflows to single column on mobile.

Accessibility: DOM order is left-to-right, top-to-bottom — logical reading order preserved for screen readers (WCAG 1.3.2). All navigation links are standard govuk-link elements with visible text labels.

{# ── Accessible chart ──────────────────────────────────────────────── #}

Accessible chart (SVG + data table)

Departure: Inline SVG charts rendered alongside govukTable() data tables, used in the Data insights section. No GDS chart component exists. Four chart types are implemented: line chart (single series), bar chart (single series), grouped bar chart (three series) and dual line chart (two series).

Justification: Charts do not exist in the GDS Design System. This implementation treats the chart as progressive enhancement: the data table carries all information and is rendered first in the DOM; the SVG is aria-hidden="true" and contributes nothing to the accessible tree. Remove the chart entirely and no information is lost.

DOM order and structure

Table is always first in DOM regardless of visual layout. Visual position (table left, chart right) is achieved with CSS grid only (WCAG 1.3.2). Each table has a govuk-visually-hidden caption for screen readers and the JS-off fallback — the tab label provides the visible heading context for sighted users.

SVG implementation

All SVGs use aria-hidden="true" on the parent <figure> and focusable="false" on the <svg> element. No role="img" is applied — it would contradict aria-hidden by assigning semantic meaning to a hidden element.

Colour and contrast compliance

All chart element colours have been audited against WCAG 1.4.11 (non-text contrast, 3:1 minimum on white background):

  • Teal #28a197 — 3.2:1. Used for line charts, bar fills, and "Actual" series. Pass.
  • Dark grey #505a5f — 7.0:1. Used for axis labels, "Forecast" series line, and "Started" bar fill. Pass.
  • Orange #f47738 — 3.1:1. Used for "Not started" bar fill (with diagonal stripe pattern). Pass.
  • Black #0b0c0c — 14.7:1. Used for legend text. Pass.
  • Gridline grey #b1b4b6 — 2.5:1. Used for decorative gridlines only — not required to meet 1.4.11 as gridlines are non-essential.

Colour independence (WCAG 1.4.1)

Multi-series charts never rely on colour alone to distinguish data:

  • Forecast vs Actual (dual line): Actual line uses solid stroke + filled circle markers. Forecast line uses dashed stroke + hollow circle markers (white fill, grey ring). At overlap points where both values are identical (e.g. Q3 and Q4 2026), both marker shapes remain visible — the hollow ring sits on top of the filled circle.
  • Housing outputs (3-bar): "Started" is solid dark grey. "Completed" is solid teal. "Not started" uses a diagonal stripe SVG pattern (orange with white stripes). Each bar also has a text label directly on the x-axis. Inline SVG legend mirrors the fill styles.
  • Single-series charts: Only one colour used — no distinction needed. Consistent data point markers (filled circles) at each value.

Progressive enhancement

When JavaScript is disabled, the govuk-tabs component falls back to showing all four panels stacked vertically. Each table retains its govuk-visually-hidden caption, and the tab text becomes an in-page anchor link — ensuring all data is accessible without any JavaScript dependency.

Windows High Contrast Mode

Chart colours are overridden by the OS in Windows High Contrast Mode. This is expected and acceptable — the data table is the primary source of information and remains fully readable. The chart is purely decorative and aria-hidden.

{# ── Government chart standards comparison ────────────────────────── #}

Comparison with government chart standards

The GOV.UK Design System has no chart component and provides no chart guidance. The closest government references are the Analysis Function chart guidance and the ONS Digital accessibility standards for data visualisation. This section documents how the implementation meets or exceeds those standards.

Government chart accessibility requirements
Requirement Source Our implementation

3:1 minimum non-text contrast WCAG 1.4.11, Analysis Function Pass — all chart element colours audited: teal #28a197 (3.2:1), dark grey #505a5f (7.0:1), orange #f47738 (3.1:1). Decorative gridlines (#b1b4b6, 2.5:1) exempt.
