Overview (GDS)

Build analysis

This page documents how the GDS-only Overview page was built. All elements use standard GOV.UK Design System components, patterns and layouts with no custom departures.

{# -- Overview --------------------------------------------------------- #}

Overview

The GDS Overview page is a pure GDS equivalent of the standard Overview page, which uses custom stat cards, inline SVG charts and CSS grid layouts. This version replaces every custom element with standard GDS components to demonstrate what is achievable without any departures from the Design System.

The page has two data states controlled by a prototype toggle in the footer:

  • MVP - heading and programme links in a single two-thirds column, no key information or data insights
  • Mature - heading in its own two-thirds row, then two-thirds (programme links) + one-third (key information) layout, plus full-width tabbed data tables

{# -- GDS alignment ---------------------------------------------------- #}

GDS components used

Every component on this page is an unmodified GOV.UK Design System component or standard GDS typography/layout class. No components have been extended, overridden or customised in any way.

  • Service navigation - govukServiceNavigation(), unmodified. Programme-level navigation (Overview, Agreed provider profile, Sites, etc.).
  • Typography - govuk-heading-xl, govuk-heading-l, govuk-heading-m, govuk-body-l, govuk-body, govuk-body-s. All standard GDS classes, unmodified.
  • Grid layout - govuk-grid-row with govuk-grid-column-two-thirds and govuk-grid-column-one-third. Standard GDS 12-column grid, unmodified.
  • Key information metrics - stacked govuk-body-s (label) above govuk-heading-m (value). Pure GDS typography classes used as a lightweight alternative to govukSummaryList for dashboard-style metrics in a narrow column.
  • Tag - govukTag() with govuk-tag--green, govuk-tag--red and govuk-tag--blue variants, unmodified. Used for trend indicators (e.g. "14% increase") in data insight tables. Text always describes the direction - colour is never the sole indicator (WCAG 1.4.1).
  • Section break - govuk-section-break--m govuk-section-break--visible between key information metrics. govuk-section-break--xl (without --visible) between the programme links section and data insights for spacing only. Standard GDS component, unmodified.
  • Tabs - govuk-tabs with data-module="govuk-tabs", unmodified. Four tab panels for data insights: Homes completed, Active sites, Housing outputs, Forecast vs Actual. Auto-initialised by govuk-frontend JavaScript. No-JS fallback shows all panels as stacked sections.
  • Table - govukTable(), unmodified. Used inside each tab panel for quarterly data. Numeric columns use format: "numeric" for right-alignment. Column headers generated by the macro with correct scope="col".
  • Summary list - govukSummaryList(), unmodified. Used in the Housing outputs tab to provide a narrative summary alongside the data table.
  • Spacing - all spacing uses govuk-!-margin-* and govuk-!-padding-* utility classes. No custom pixel values anywhere on the page.
  • Font size overrides - no custom font sizes. All text uses the sizes provided by GDS typography classes (govuk-heading-m for metric values, govuk-body-s for labels).
  • Links - govuk-link class on all anchor elements. Programme section links use govuk-heading-m wrapping a govuk-link, following the standard GDS pattern for navigation sections.
  • Lists - govuk-list for programme section child links. Unmodified.
  • Details - govukDetails(), unmodified. Used for the prototype controls toggle in the footer.
  • Select - govukSelect(), unmodified. Used inside prototype controls for data state switching.
  • Button - govukButton() with govuk-button--secondary, unmodified. Used for the prototype controls Apply button and Build analysis link.
{# -- Custom departures ------------------------------------------------ #}

Custom departures

None. Every element on the GDS Overview page uses standard GOV.UK Design System components and classes. There are no custom CSS classes, no custom Sass, no inline styles, no custom JavaScript and no SVG graphics.

This is the key difference from the standard Overview page, which uses custom stat cards (.stat-card-grid), inline SVG charts (aria-hidden="true"), custom insight blocks (.insight-block) and a custom CSS grid layout (.nav-section-grid). The GDS version achieves the same information architecture using only components from the Design System.

{# -- Design trade-offs ------------------------------------------------ #}

Design trade-offs

Staying within pure GDS involves trade-offs compared to the custom version. These are documented here for design review.

Charts replaced with data tables

The standard Overview uses inline SVG line charts and bar charts to show trends visually. The GDS version replaces these entirely with govukTable and govukTag. The data is identical but the visual impact is reduced - users must read numbers and percentage tags rather than scanning a chart shape. For users who are comfortable with tabular data (SP users typically are), this may be acceptable. For users who rely on visual pattern recognition, it is a regression.

Stat cards replaced with stacked typography

The standard Overview uses bordered metric cards in a 2x3 grid to give six key metrics high visual prominence. The GDS version uses stacked govuk-body-s / govuk-heading-m pairs in a one-third sidebar column. This occupies less horizontal space but makes the metrics less scannable - they are read vertically rather than in a grid. The section break dividers help separate each metric visually.

Two-column programme links vs single-column

The standard Overview uses a CSS grid to show programme sections in two columns. The GDS version uses a single-column list in a two-thirds column. This is more conservative and aligns with GDS guidance on content pages, but creates a longer vertical scroll for the programme links section.

{# -- Accessibility ---------------------------------------------------- #}

Accessibility

Because every component on this page is a standard, unmodified GOV.UK Design System component, the accessibility characteristics are inherited directly from the Design System. GOV.UK Frontend components are tested against WCAG 2.2 AA by the GDS team and undergo regular accessibility audits.

No custom accessibility testing is required beyond the standard testing that would apply to any page using GDS components:

  • Run axe DevTools to confirm zero violations - expected to pass as all components are standard GDS
  • Run Lighthouse accessibility audit - expected to score 100 as there are no custom elements
  • Verify keyboard navigation - all GDS components are natively keyboard accessible
  • Verify screen reader announcements - GDS tabs, tables and tags all have built-in ARIA support

The detailed 12-row assistive technology testing matrix used for the custom Overview page and Sites page is not needed here. That matrix exists to validate custom departures (editable tables, stat cards, SVG charts, custom ARIA patterns). When all components are unmodified GDS, the testing burden falls on the GDS team's own component tests, not on the service team.

The one area worth confirming manually is the stacked label/value pattern in the key information sidebar. While it uses only standard GDS typography classes (govuk-body-s, govuk-heading-m), the composition of multiple metrics with section breaks is a layout decision rather than a documented GDS pattern. A quick screen reader check should confirm that each label-value pair is announced clearly in sequence.
